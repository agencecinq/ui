## Install

```bash
pnpm add @agencecinq/tabs
```

## Usage

Import once:

```js
import '@agencecinq/tabs';
```

Then write the WAI‑ARIA markup:

```html
<cinq-tabs data-tabs-hash="false" data-tabs-delay="0">
  <div role="tablist" aria-label="Example tabs">
    <button id="tab-home" role="tab" aria-selected="true" aria-controls="panel-home">
      Home
    </button>
    <button id="tab-project" role="tab" aria-selected="false" aria-controls="panel-project" tabindex="-1">
      Project
    </button>
  </div>

  <section id="panel-home" role="tabpanel" aria-labelledby="tab-home" tabindex="0">…</section>
  <section id="panel-project" role="tabpanel" aria-labelledby="tab-project" tabindex="0" hidden>…</section>
</cinq-tabs>
```

## Options

Configure via data attributes:

- **`data-tabs-hash`**: `true` (default) / `false` — sync the active tab id to `location.hash` on activation.
- **`data-tabs-delay`**: number (ms), default `0` — when `> 0`, arrow key navigation activates after a delay.

## API

### Method

- **`activateTab(index: number)`**: activates a tab programmatically (useful after canceling `tab-before-activate`).

### Events

- **`tab-before-activate`** (cancelable): `detail: { index, controls, element }`
- **`tab-activate`**: `detail: { controls, element }`
- **`tab-delete`**: `detail: { controls, element }`

### RTL

If the tablist inherits `direction: rtl` (e.g. `dir="rtl"`), left/right arrow navigation follows the reading direction.

```html
<div dir="rtl">
  <cinq-tabs data-tabs-hash="false">…</cinq-tabs>
</div>
```
